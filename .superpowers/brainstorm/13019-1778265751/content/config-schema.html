Config Schema

YAML in ConfigMap klyra-config — the full shape of what a monitor definition looks like

monitors (k8s example)

monitors:
  - name: api-deployment-health
    type: kubernetes
    interval: 30s
    config:
      kind: deployment
      namespace: production
      name: api
      check: ready_replicas
    threshold:
      operator: lt
      value: 2
      for: 2m   # must stay true for 2m to fire
    actions:
      - notify-ops
      - notify-oncall

  - name: node-not-ready
    type: kubernetes
    interval: 60s
    config:
      kind: node
      check: ready_condition
    threshold:
      operator: eq
      value: false
      for: 5m
    actions:
      - notify-ops

monitors (http example)

  - name: api-health-check
    type: http
    interval: 15s
    config:
      url: https://api.example.com/health
      method: GET
      timeout: 5s
      expect_status: 200
      expect_body: "ok"      # optional substring match
      headers:
        Authorization: Bearer ${SECRET_TOKEN}
    threshold:
      operator: eq
      value: false    # fires if check fails
      for: 1m
    actions:
      - notify-ops

actions

actions:
  - name: notify-ops
    type: http
    config:
      url: https://ntfy.sh/ops-alerts
      method: POST
      auth:
        type: bearer
        token: ${NTFY_TOKEN}
      ntfy:
        priority: high       # urgent|high|default|low|min
        tags: [warning, k8s]

  - name: notify-oncall
    type: http
    config:
      url: https://ntfy.sh/oncall-pager
      method: POST
      auth:
        type: bearer
        token: ${NTFY_TOKEN}
      ntfy:
        priority: urgent
        tags: [rotating_light, oncall]

kubernetes checks (supported)

kindcheck
deploymentready_replicas, available_replicas, paused
podphase, restarts, ready_condition
nodeready_condition, disk_pressure, memory_pressure
eventreason (match string/regex), type=Warning

threshold operators

operatormeaning
lt / gtnumeric less/greater than
lte / gtenumeric ≤ / ≥
eq / neqequal / not equal (numeric or bool)
containsstring substring match
matchesregex match

for: 2m — condition must hold for duration before firing. Optional (default: fire immediately)
